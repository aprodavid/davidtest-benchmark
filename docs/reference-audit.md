# Reference Audit — bareun.vercel.app

## 0) 근거 및 방법
- **1차 근거(우선):** 사용자 제공 모바일 스크린샷 10장
- **2차 근거(보조):** 공개 URL `https://bareun.vercel.app/` 직접 확인 시도
  - `web open`으로 URL 도달 자체는 확인됨
  - `curl` 직접 조회는 `CONNECT tunnel failed, response 403`으로 본문 확인 실패
- 본 문서는 **공개적으로 보이는 UI와 동작만** 정리하며, 코드 구현 추정은 별도 구분한다.

---

## 1) 화면 목록
식별 가능한 화면은 다음과 같다.

1. 메인(홈): **체육관 물품 관리**
2. 물품 대여하기 - 1단계: **대여 물품 선택(다중 선택)**
3. 물품 대여하기 - 2단계: **물품별 수량 조정**
4. 물품 대여하기 - 3단계-A: **대여자 정보 입력(학년/반 선택 탭)**
5. 물품 대여하기 - 3단계-B: **대여자 정보 입력(직접 입력 탭)**
6. 물품 반납하기: **대여중 항목 선택 후 반납 완료**
7. 관리자 인증
8. 관리자 모드(대시보드)
9. 물품 마스터 관리
10. 전체 대여 기록

---

## 2) 화면별 주요 UI 요소

### 2.1 메인(홈)
- 상단 우측: 톱니바퀴 아이콘(관리자 진입 추정)
- 중앙 타이틀: `체육관 물품 관리`
- 부제: `쉽고 빠른 대여/반납 시스템`
- 2개 대형 CTA 카드
  - 녹색 `대여하기`
  - 빨간색 `반납하기`

### 2.2 대여 1단계(물품 선택)
- 헤더: 뒤로가기 + `물품 대여하기`
- 안내문: `대여할 물품을 선택해주세요 (다중 선택 가능)`
- 그리드 카드(3열 배치로 보임)
  - 품목 이미지, 품목명, `남은 수량: N개`
  - 선택 시 연녹색 배경/초록 테두리/체크 배지
- 하단 고정 버튼: `다음 단계로`

### 2.3 대여 2단계(수량 조정)
- 헤더: 뒤로가기 + `물품 대여하기`
- 안내문: `대여할 수량을 확인 및 조정해주세요`
- 품목 카드 리스트
  - 좌: 이미지 + 품목명 + `최대 N개 가능`
  - 우: 수량 스텝퍼(`- 1 +`) 또는 상태 배지
  - 예: `수량 입력 없음 (1)`
- 하단 고정 버튼: `다음 단계로`

### 2.4 대여 3단계-A(학년/반)
- 헤더: 뒤로가기 + `물품 대여하기`
- 안내문: `누가 빌리는지 알려주세요`
- 탭: `학년 / 반 선택` / `직접 입력`
- 드롭다운 2개
  - `학년 (GRADE)`
  - `반 (CLASS)`
- 대여 요약 박스(연녹색)
  - 선택 품목/수량 bullet 표시
- 하단 고정 버튼: `대여 완료하기`

### 2.5 대여 3단계-B(직접 입력)
- 헤더/안내문/요약/하단 버튼 구성은 3단계-A와 동일
- 탭 상태: `직접 입력` 활성
- 입력 영역
  - 라벨: `이름 또는 부서 입력`
  - placeholder: `예) 홍길동 선생님, 학생회`

### 2.6 반납 화면
- 헤더: 뒤로가기 + `물품 반납하기`
- 안내문: `현재 대여 중인 항목입니다. 반납할 항목을 선택하세요.`
- 대여중 카드(가로 2열 배치로 보이나 화면폭에 따라 변할 수 있음)
  - 좌측 원형 선택 컨트롤(라디오/체크 형태)
  - 품목 이미지, 품목명, 수량(빨간색 강조)
  - 대여자 정보(학년/반), 대여일시
- 하단 고정 버튼
  - 선택 0건: 비활성 느낌 `선택한 0건 반납 완료`

### 2.7 관리자 인증
- 헤더: 뒤로가기 + `관리자 인증`
- 중앙 인증 카드
  - 아이콘 + 타이틀
  - 안내: `비밀번호를 입력해주세요. (초기 비밀번호: 0000)`
  - 입력창: `비밀번호 입력`
  - 버튼: `확인`

### 2.8 관리자 모드
- 헤더: 뒤로가기 + `관리자 모드`
- 메뉴 카드
  - `물품 마스터 관리` (새 교구 추가/삭제)
  - `전체 대여 기록` (히스토리 조회)
  - `관리자 설정` (관리자 비밀번호 변경)

### 2.9 물품 마스터 관리
- 헤더: 뒤로가기 + `물품 마스터 관리`
- 섹션 1: `새 물품 추가`
  - 사진 업로드 박스
  - 물품명 입력 (`예: 배드민턴 채`)
  - 총 보유 수량 입력(숫자)
  - 체크박스: `수량 선택 기능 켜기 (체크 해제 시 1개 고정)`
  - 버튼: `추가하기`
- 섹션 2: `등록된 물품 목록`
  - 품목 카드: 이미지, 이름, 총 수량
  - 우측 삭제 아이콘(휴지통)

### 2.10 전체 대여 기록
- 헤더: 뒤로가기 + `전체 대여 기록`
- 설명문: `모든 대여 및 반납 기록입니다.`
- 기록 카드 리스트
  - 상태 배지: `대여중` 또는 `반납완료`
  - 우측 상단: 날짜/시간
  - 품목명 + 수량
  - 대여자(학년/반 혹은 사용자 식별 정보)
  - `대여중` 항목은 녹색 강조 테두리

## 3) 섹션/카드/버튼/입력창 구성 요약

### 섹션 패턴
- 상단 앱바(뒤로가기 + 타이틀)
- 본문 카드 섹션(라운드 + 연회색 배경)
- 상태 강조 섹션(연녹색 배경: 요약/선택/활성)
- 하단 고정 액션 바(주요 CTA)

### 카드 패턴
- 리스트 카드(행 기반): 이미지 + 텍스트 + 우측 액션
- 그리드 카드(열 기반): 이미지 중심 + 라벨 + 수량
- 상태 카드: 배지/색상으로 진행 상태 전달

### 버튼 패턴
- Primary(녹색): 다음 단계/완료
- Secondary(남색): 확인(관리자 인증)
- Disabled(회색): 조건 미충족 시 비활성
- Icon-only: 삭제/설정

### 입력 패턴
- 탭 전환형 입력(학년/반 vs 직접입력)
- 드롭다운(학년, 반)
- 텍스트 인풋(이름/부서, 물품명, 관리자 비밀번호)
- 스텝퍼(수량 +-)
- 체크박스(수량 선택 기능 on/off)

## 4) 사용자 흐름

### 4.1 일반 사용자 — 대여
1. 홈 `대여하기` 선택
2. 대여 1단계에서 물품 다중 선택
3. 대여 2단계에서 품목별 수량 조정
4. 대여 3단계에서 대여자 정보 입력
   - (A) 학년/반 선택 또는 (B) 직접 입력
5. `대여 완료하기` 실행
6. 이후 반납 화면/전체 기록에 대여중 상태 반영

### 4.2 일반 사용자 — 반납
1. 홈 `반납하기` 선택
2. 대여중 항목에서 반납 대상 선택
3. `선택한 N건 반납 완료` 실행
4. 기록에서 상태가 `반납완료`로 변경

### 4.3 관리자
1. 홈 우측 상단 설정(톱니) 진입
2. 관리자 인증 화면에서 비밀번호 입력
3. 관리자 모드에서 목적 기능 선택
   - 물품 마스터 관리
   - 전체 대여 기록 조회
   - 관리자 설정(비밀번호 변경)

## 5) 모바일 기준 레이아웃 특성
- 세로형 단일 컬럼 스크롤 중심
- 주요 액션 버튼은 **하단 고정(full-width)**
- 카드 기반 정보 구조(터치 타겟 큼)
- 색상으로 상태 구분(녹색=진행/가능, 회색=비활성/완료, 빨강=반납/강조)
- 일부 선택/목록 화면은 2열~3열 그리드 사용

---

## 6) 화면 간 이동 관계

- 홈
  - `대여하기` → 대여 1단계
  - `반납하기` → 반납 화면
  - `설정(톱니)` → 관리자 인증
- 대여 1단계 → 2단계 → 3단계 → 완료
- 관리자 인증 → 관리자 모드
- 관리자 모드
  - `물품 마스터 관리` ↔ (뒤로) 관리자 모드
  - `전체 대여 기록` ↔ (뒤로) 관리자 모드
  - `관리자 설정` → 비밀번호 변경 화면(미확보)

---

## 7) 확실한 내용
- 앱의 핵심 기능은 **대여/반납** 2축으로 구성됨.
- 관리자 영역은 별도 인증을 요구함.
- 대여는 최소 3단계(선택→수량→대여자 정보)로 보임.
- 반납은 대여중 항목 선택 후 일괄 처리 구조를 가짐.
- 물품 마스터 관리에서 품목 등록/삭제가 가능함.
- 전체 대여 기록에서 `대여중`/`반납완료` 상태가 구분됨.

## 8) 추정한 내용
- 대여 완료 직후 성공 토스트/확인 다이얼로그가 있을 가능성.
- 반납 완료 후 즉시 목록 재조회 및 상태 갱신이 일어날 가능성.
- 관리자 설정 메뉴에서 비밀번호 변경 폼(현재/신규/확인)이 제공될 가능성.
- 데스크톱에서도 동작하되 모바일 폭 최적화 레이아웃을 유지할 가능성.

## 9) 불확실한 내용
- URL 경로 체계(예: `/rent/step-1` 형태) 및 라우팅 방식
- 유효성 검사 상세 규칙(최소/최대 수량, 공백 입력 처리 등)
- 오류 처리 UX(네트워크 실패, 인증 실패, 재시도)
- 반납 화면 선택 컨트롤의 정확한 컴포넌트 타입(원형 체크 UI만 확인 가능)
- 기록 정렬 기준(최신순 고정 여부)

## 10) 추가로 필요한 화면 캡처 목록
정확한 IA/플로우 문서 완성도를 높이기 위해 아래 캡처가 추가로 필요함.

1. 대여 1단계에서 **미선택 상태/선택 완료 후 버튼 활성 변화** 비교
2. 대여 2단계에서 **여러 품목 수량 입력 완료 상태**
3. 대여 3단계에서 **학년/반 드롭다운 펼침 상태**
4. 대여 완료 직후 **성공 피드백 화면(토스트/모달/리다이렉트)**
5. 반납 화면에서 **항목 1개 이상 선택된 상태(버튼 활성)**
6. 반납 완료 직후 **성공 피드백 및 목록 갱신 상태**
7. 관리자 인증 **실패 상태(오류 메시지)**
8. 관리자 설정 진입 후 **비밀번호 변경 화면 전체**
9. 물품 마스터 관리에서 **신규 물품 추가 성공 상태**
10. 전체 대여 기록의 **필터/검색/페이징 존재 여부**
