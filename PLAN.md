# PLAN

## 작업 목표
첨부된 10장의 모바일 스크린샷을 1차 근거로 사용해 `https://bareun.vercel.app/` 앱의 화면 구조와 사용자 흐름을 문서화한다.

## 범위
- 작성 파일
  - `PLAN.md`
  - `docs/reference-audit.md`
- 제외 범위
  - 코드 구현/수정
  - 기능 개발
  - 테스트/배포

## 접근 방식
1. 스크린샷 10장을 화면 단위로 분류한다.
2. 각 화면의 UI 구성요소(섹션/카드/버튼/입력창)를 정리한다.
3. 화면 간 이동 관계와 사용자 흐름을 단계별로 정리한다.
4. 확실/추정/불확실 정보를 분리해 표기한다.
5. 공개 사이트 직접 확인 결과는 보조 근거로 병기한다.

## 산출물 기준
`docs/reference-audit.md`에 아래 항목을 반드시 포함한다.
- 화면 목록
- 화면별 주요 UI 요소
- 섹션/카드/버튼/입력창 구성
- 사용자 흐름
- 모바일 기준 레이아웃 특성
- 화면 간 이동 관계
- 확실한 내용
- 추정한 내용
- 불확실한 내용
- 추가로 필요한 화면 캡처 목록

## 현재 확인 상태
- 스크린샷 10장: 확보됨
- 공개 사이트 직접 확인: 제한적(네트워크/렌더링 제약으로 정적 검증만 시도)
